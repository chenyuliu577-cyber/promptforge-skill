# Report Writing Example

## User's Original Simple Request

帮我写实验报告。

## Task Analysis

- Primary task type: writing
- Secondary task type: documents
- Objective: produce a structured report-writing prompt rather than generic prose
- Missing data: experiment purpose, data, required sections, citation rules, word count

## Key Assumptions

- a standard lab report structure is acceptable if the user gives no template
- factual accuracy matters more than stylistic flourish
- unsupported data must never be invented

## Optimized Final Prompt

```text
Write a structured experiment report based only on the information provided.

Objective:
Produce a clear academic-style report suitable for submission or revision.

Requirements:
- use a standard structure: objective, method, materials, procedure, results, analysis, conclusion
- clearly separate observed results from interpretation
- identify any missing data or unsupported claims instead of inventing them
- maintain formal, concise language
- if the instructor format is unknown, note that the structure is a reasonable default
- quality standard: every conclusion must be traceable to provided experiment data

Do not:
- fabricate measurements, citations, or conclusions
- blur the line between data and explanation

Verification Checklist:
- all sections are present
- analysis follows from the provided results
- missing evidence is flagged explicitly
```

## Risk Check

- do not invent measurements, observations, citations, or instructor requirements
- flag missing data instead of filling gaps with unsupported claims
- avoid submitting or presenting the report as final without user review

## User Confirmation Point

No special confirmation is required for a draft. Ask for confirmation before formatting for submission, sending, uploading, or claiming the report is final.

## Checklist

- report structure is controlled
- evidence fabrication is blocked
- missing information is surfaced
- academic tone is specified
- confirmation status is explicit
